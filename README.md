# Maritime Risk Series

A data-driven actuarial and operational study of maritime operational risk, recovery systems, and ship–infrastructure exposure.

## About

This repository documents a series of maritime risk projects built at the intersection of:

- seafaring and marine engineering experience,
- commercial shipping operations,
- actuarial thinking,
- and R/Python-based analysis.

The aim is to study how operational failures such as blackouts translate into risk, exposure, and loss.

## Series outline

1. Blackout recovery and the Dali context
2. What a blackout actually means onboard
3. Load-sharing vs emergency generator fallback
4. Exposure, restricted waters, and operational context
5. Frequency, severity, and tail risk

## Repository structure

- `data/` raw and processed datasets
- `notebooks/` project notebooks and analysis files
- `scripts/` R scripts for simulation and plotting
- `outputs/figures/` graphs and exported visuals
- `posts/` LinkedIn drafts for each part
- `references/` source notes and methodology

## Tools used

- R
- Python
- Google Sheets
- Markdown

## Limitations

Some models in this repository are scenario-based and informed by regulation, public information, and operational experience. They are exploratory analyses, not underwriting tools.

## Why this matters
This project is an attempt to connect onboard reality with actuarial reasoning, and to think more carefully about maritime risk as a measurable system rather than only as a headline event.
